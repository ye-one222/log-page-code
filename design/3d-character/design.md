# 3D 캐릭터 디자인 문서 — Minecraft 스킨 스타일 페르소나

> 개인 웹사이트(Astro) 홈 히어로의 회전-변신 캐릭터를 2D SVG에서 3D 복셀 캐릭터로 교체하기 위한 디자인 사양.
> 구현: Three.js `BoxGeometry` + 캔버스로 그린 픽셀 텍스처(`NearestFilter`). 지름 230px 원형 스테이지 안에 렌더.
> 하나의 캐릭터 = 스킨 4벌. 얼굴/피부는 공유, 옷·머리·소품으로 페르소나를 구분한다.

---

## 0. 설계 원칙

- **저해상도 픽셀 우선**: 모든 표면은 Minecraft 스킨 포맷(64×64 텍스처)을 따르고 확대 시 `NearestFilter`로 픽셀이 또렷하게 보이게 한다. 안티앨리어싱·부드러운 그라데이션 금지.
- **단색 금지, 디더링 필수**: 각 면은 베이스 톤 1개 + 근접 톤 1~3개를 픽셀 단위로 섞어 옷감/머리결/니트 질감을 만든다(§4 디더링 규칙).
- **관절 없음**: 순수 Minecraft 비율. 팔·다리는 막대(box) 하나씩, 곡면·라운드 없음.
- **소품도 복셀**: 안경·카메라·책·기타까지 전부 작은 박스의 조합. 원기둥·구 금지.
- **피부/얼굴 공유**: 4벌 모두 동일한 head 텍스처(피부 + 눈/입)를 쓰고, 그 위 머리카락 overlay 레이어와 옷/소품만 페르소나별로 교체한다.

## 1. 파츠 박스 치수·배치 (Minecraft 표준 비율)

단위: **Minecraft 픽셀(1u)**. 표준 스킨과 동일하게 캐릭터 전체 높이 = 32u.
Three.js 월드 스케일은 `1u = 0.1 world unit` 권장(전체 3.2 world height). 스테이지에 맞춰 카메라/그룹 스케일로 230px 안에 채운다.

| 파츠 | 크기 (W×H×D, u) | 중심 위치 (x, y, z) | 비고 |
|---|---|---|---|
| Head | 8 × 8 × 8 | (0, 24, 0) | 정육면체. 눈높이 y≈25 |
| Body(Torso) | 8 × 12 × 4 | (0, 14, 0) | 납작한 직육면체 |
| Right Arm | 4 × 12 × 4 | (−6, 14, 0) | 몸통 좌우 밀착(x=±6) |
| Left Arm | 4 × 12 × 4 | (+6, 14, 0) | |
| Right Leg | 4 × 12 × 4 | (−2, 2, 0) | 두 다리 붙임(x=±2) |
| Left Leg | 4 × 12 × 4 | (+2, 2, 0) | |

바닥선 y=0(발밑), 머리 꼭대기 y=28 → 머리카락/소품 볼륨 포함해 스테이지 세로 대비 약 85% 채움.

좌표계: **+y 위, +z 정면(카메라 방향), +x 캐릭터의 왼쪽**. 회전은 group 전체를 y축(`rotation.y`) 기준으로 돈다.

### 카메라 / 스테이지
- **정면 고정**, 살짝 위(약 −8° pitch)에서 내려다봄. 회전은 캐릭터 group만 돈다.
- 배경 투명(alpha:true). 원형 스테이지는 기존 `.ring`의 CSS 원(230px)이 그대로 마스크 역할.
- 라이트: `HemisphereLight`(하늘 `#FFF9F2` / 바닥 `#E7DCCB`, 강도 0.9) + 정면 위 `DirectionalLight`(강도 0.35). 픽셀 텍스처가 주인공이므로 `MeshLambertMaterial`로 톤 유지, 셀프 섀도우 off.

## 2. 텍스처 아틀라스 — Minecraft 64×64 포맷 채택

캐릭터당 캔버스 1장(64×64px). 6면 UV를 표준 Minecraft 레이아웃대로 배치한다(px, 좌상단 원점):

| 영역 | UV 박스 (x, y, w, h) |
|---|---|
| Head 전면 | (8, 8, 8, 8) |
| Head 후면 | (24, 8, 8, 8) |
| Head 우/좌/상/하 | (0,8)/(16,8)/(8,0)/(16,0) 각 8×8 |
| Hat/Overlay(머리카락) | Head UV +32px 오프셋(예 40,8), 동일 6면 |
| Body 전면 / 후면 | (20,20,8,12) / (32,20,8,12) |
| Right Arm 전면 | (44, 20, 4, 12) |
| Left Arm 전면 | (36, 52, 4, 12) |
| Right Leg 전면 | (4, 20, 4, 12) |
| Left Leg 전면 | (20, 52, 4, 12) |

구현 팁:
- 캔버스 `imageSmoothingEnabled = false`. Three 텍스처 `minFilter = magFilter = THREE.NearestFilter`, `generateMipmaps = false`.
- BoxGeometry 6면 UV를 위 좌표로 수동 매핑(`geometry.attributes.uv` 편집). 유틸 `setBoxUV(geometry, faceUVMap)` 하나로 처리.
- 머리카락은 head보다 살짝 큰 box(8.5³~9³ u)에 overlay 텍스처를 입혀 겉머리 레이어처럼 덮는다. 안 덮이는 픽셀은 알파.

## 4. 디더링 규칙 (질감의 핵심)

각 면은 8×8(head/body 전면) 또는 4×12(팔/다리) 픽셀 그리드. **단색 채움 금지.**

1. **2톤 체커 디더**: 베이스색 B + 근접 음영 S를 2×2 체커로 깔되, 면 전체의 약 15~25%만 S로 흩뿌린다(전부 체커면 노이지). 옷감 기본 질감.
2. **결(grain) 방향**: 머리카락은 세로 1px 줄무늬(밝은 하이라이트 세로), 니트(독서인)는 가로 2px 줄무늬로 짜임 표현.
3. **하이라이트 스팟**: 어깨/머리 위 등 광원 닿는 곳에 light 톤 2~4픽셀 클러스터.
4. **테두리 음영**: 각 면 가장자리 1px는 shade 톤으로 감싸 볼륨감(픽셀 아웃라인 대용).
5. 근접 톤은 베이스에서 명도 ±8~12% 범위. 채도 유지, 색상은 거의 안 바꿈.

예: 흰 상의 → base `#FFFFFF`, shade `#ECECEC`, 그림자 `#DADADA` 3톤을 위 규칙으로 섞음.

## 5. 페르소나별 스킨 사양

공통: head 피부 텍스처 + 눈(정면 하단 2×2 ink, 좌우), 입(1×3 ink 살짝 웃음), 볼 blush 1px. 머리 실루엣은 단발(head를 감싸는 overlay, 앞머리는 이마 위 2u 내려옴).

### 5.1 연구원 (researcher · 보라 `#7F77DD`)

- **머리카락**: 로즈 베이스 + 보라 `#3C3489` 세로 하이라이트 몇 픽셀.
- **소품 A — 픽셀 안경**: 렌즈 2개(각 4×4×1u 얇은 box, 알파 유리 `#EAF0FF` + ink 테두리), bridge 2×1×1u ink, head 전면 z=+4.1에 부착(눈 위에 겹침).

### 5.2 크리에이터 (creator · 청록 `#1D9E75`) — 멜빵바지(overalls)

## 6. 회전-변신 연출

기존 `PersonaHero.astro`는 CSS로 “`rotateY(90deg)` → 스킨 교체 → `-90deg` 순간이동 → `0deg`”의 관통 회전(300ms, `ease`)을 만든다. 3D 캐릭터는 이를 **Three group의 `rotation.y` 직접 애니메이션**으로 재현한다.

버튼 클릭 1회 = 반 바퀴 관통:
1. **0 → 150ms**: `rotation.y` 0 → +90°(옆면이 카메라로). `easeInOutCubic` 가속. 옆면(깊이 4u)이 얇아 자연스레 사라짐.
2. **≈90° 통과 순간(가장 얇을 때)**: 이 프레임에서 **스킨 스왑**(다음 페르소나 텍스처 `material.map` 교체 + 소품 group 교체). “뒤돌아보니 다른 사람” 효과.
3. **150 → 300ms**: `rotation.y` +90° → +180°(정면, `mod 360`으로 0 리셋). 감속. 새 페르소나가 정면 등장.
4. 포인트 컬러(ring border, dots)는 기존 스크립트 담당 — 3D는 텍스처/소품만.

세부:
- 총 **300ms**, 기존 CSS 타이밍과 동기. `busy` 잠금 유지(연타 방지).
- 스왑은 `rotation.y`가 88~92° 구간일 때 `swapped` 플래그로 1회만.
- 소품은 페르소나별 `THREE.Group`을 미리 만들어 `visible` 토글(생성 비용 절약).
- **prefers-reduced-motion**: 회전 없이 즉시 텍스처만 교체(기존 스크립트 분기와 동일).

## 7. Idle 애니메이션 (회전 안 할 때)

전부 미세하게, “살아있는 정지” 정도. `THREE.Clock` 기반, 회전 중(`busy`)엔 일시정지.

1. **부유(bob)**: group `position.y`를 `sin(t·1.2)·0.05u` 진폭으로 상하. 주기 ≈ 5초.
2. **호흡(breathe)**: body/arm `scale.y` ±1% `sin(t·0.8)`. 아주 약하게.
3. **머리 갸웃**: head `rotation.z` `sin(t·0.5)·1.5°`, `rotation.y` `sin(t·0.35)·3°`로 미세하게 둘러봄.
4. **깜빡임(blink)**: 4~7초 랜덤 간격마다 눈 텍스처를 2프레임(≈120ms) “감은 눈”(1×2 ink 가로선)으로 스왑 후 복귀. 눈 부분만 별도 작은 plane 또는 텍스처 2장.
5. **소품 반응**: 소품 group이 부유에 살짝 뒤따라 흔들리게 `position.y` 지연 lerp(관성).

성능: 60fps 목표(폴리곤 매우 적음). `renderer.setPixelRatio(min(devicePixelRatio, 2))`, 탭 비활성 시 `requestAnimationFrame` 정지.
